## 1. Why：为什么要做这个 POC

### 1.1 问题陈述

第一代「基地」产品验证了校友网络的价值，但暴露了一个核心痛点：**大多数校友不愿意花时间填写个人主页**。手动填写表单枯燥、标签选择困难、简介撰写痛苦 —— 结果是大量空白或敷衍的个人档案，让整个网络的互赖匹配能力大打折扣。

### 1.2 核心假设

| # | 假设 | 验证指标 | 成功标准 |
|---|------|---------|---------|
| H1 | 用户更愿意「口述录入」而非「手动填表」来创建个人主页 | 口述创建完成率 vs 手动填写完成率 | 口述路径完成率 ≥ 70% |
| H2 | AI 采访式引导能挖掘出用户自己没想到的标签 | AI 新增标签占比（用户未主动提及的标签数 / 总标签数） | AI 新增标签占比 ≥ 30% |
| H3 | 结构化内容块 + 严格增量自动更新，能让主页「越聊越准」且用户始终拥有绝对控制权 | 用户对 AI 生成核心信息的**实质性纠偏/删改比例** | 实质性纠偏率 ≤ 30%（说明 AI 足够准） |
| H4 | 产品经理无需改代码即可调优 AI 效果 | Prompt 修改到生效的时长 | 从修改到生效 < 5 分钟 |

> **v1.2 变更说明**：
> - H3 假设描述强化：「自动更新标签」→「严格增量自动更新」，明确用户始终拥有绝对控制权。与故事四的「用户是 Boss」增量机制呼应。

### 1.3 POC 成功标准

POC 通过 = 以下 **3 项同时达标**：

1. **3 位以上测试用户**能独立完成口述或采访创建个人主页的完整流程
2. **AI 提取的标签**中 ≥ 30% 是用户未主动提及的（验证 H2）
3. **产品经理**能在 5 分钟内通过后台修改 Prompt 并看到效果变化（验证 H4）

## 3. What：功能规格

### 3.1 用户端功能

#### 故事一：口述创建个人主页（P0）

> **作为** 一位新注册的校友，
> **我想要** 通过口述自我介绍（使用系统键盘语音输入），让 AI 自动转化为恰当的结构化标签和一段描述文字，并生成一张个人展示图片，
> **以便** 我能快速、轻松地创建专业且真实的个人主页，无需费力排版。

**验收标准：**
- Given 用户在创建个人主页界面
- When 用户在输入框中键入或口述自我介绍内容（通过系统键盘的语音转文字功能）
- Then AI 提取标签并生成一段描述文字，同时生成一张个人展示图片

**交互细节：**
- 输入方式为**纯文字交互**，系统本身不处理任何语音流
- 输入框 Placeholder 明确提示：「试试用手机键盘的语音输入，口述你的自我介绍」
- 输入框旁显示语音输入引导图标（非录音按钮），点击弹出系统键盘语音输入的简短说明
- 文字输入后，AI 意图舱进入 Parsing 态（琥珀金跑马灯）→ Field_Filling 态（逐字段琥珀金浅底闪烁填充）
- 支持长文本输入和多段输入
- **大文本输入框行为**：输入框随内容自动撑高，挤压下方预览区域空间。**禁止内部滚动**——用户不需要在输入框内上下移动光标或滚动查看，交互保持简单直观。下方预览区域随输入框撑高而自然收缩，但保持最小可读高度

> **v1.3 变更说明**：新增大文本输入框自动撑高规格，明确禁止内部滚动，挤压下方空间而非出现滚动条。

#### 故事二：AI 对话式采访引导（P0）

> **作为** 一位校友，
> **我想要** AI 以对话形式采访我，引导我说出背景、能力和需求，
> **以便** 挖掘出我自己可能都没意识到的重要标签和特质，让主页更真实完整。

**验收标准：**
- Given 用户处于 AI 采访模式
- When AI 提出引导性问题（如「你最擅长什么？」「能给大家提供什么帮助？」）
- Then 用户通过文字回答，AI 将对话内容结构化为可编辑的内容块
- And 当采访满足收敛条件时，AI 主动宣布采访完成并生成主页；用户也可随时手动结束采访

**采访收敛条件（三重保障）：**

| 条件 | 规则 | 说明 |
|------|------|------|
| 维度覆盖 | 集齐 self_intro + background + offer + need 四个维度的内容块 | 后台 Prompt 定义目标维度，AI 自检已有维度覆盖度 |
| 最大轮数 | 采访对话不超过 10 轮（可后台配置） | 防止无限追问 |
| 用户主动退出 | 常驻「提前结束采访并生成主页」按钮 | 用户随时可结束 |

**交互细节：**
- 采访界面为左右分栏：左侧对话面板（AI 提问 + 用户回答气泡），右侧实时预览内容块和标签
- AI 提问气泡带分类色标（自我介绍/背景/能提供/需要）
- 对话采用 SSE 流式输出，用户可随时打断或跳过
- AI 根据已有内容判断是否追问（追问逻辑可后台配置）
- **常驻退出按钮**：对话面板顶部或底部固定显示「结束采访，生成我的主页」按钮
- **进度指示**：显示当前采访维度覆盖进度（如「已覆盖 2/4 维度」）
- **采访结束时**：AI 生成一段总结性文字 + 完整标签 + 简介建议，进入主页预览态

**异常与中断恢复：**
- **SSE 连接中断**：前端自动重试（最多 3 次，间隔 2s），重试期间显示「正在重新连接…」提示，已接收的内容块和标签不丢失
- **AI API 超时**（> 30s）：显示「AI 正在思考，请稍候…」，超时后提示「响应超时，请重试」，已生成的部分内容保留
- **API 安全拦截**（内容审核触发）：显示友好提示「当前输入无法处理，请调整内容后重试」，不暴露安全拦截细节
- **前端兜底**：所有 SSE 流式输出支持断点续传，页面刷新后通过已有内容块状态恢复上下文

#### 故事三：结构化内容块的增删改（P0）

> **作为** 一位校友，
> **我想要** 我的自我介绍内容按「块」保存（如自我介绍、历史背景、能提供的东西、具体需求），并可以随时新增内容块，也可以手动删除和编辑块
> **以便** 我能持续、灵活地完善我的个人档案，无需一次性写完。

**验收标准：**
- Given 用户已存在个人主页
- When 用户点击「新增内容块」并输入内容（如「我大学时是物理竞赛省一」）
- Then 系统保存该内容块，用户可点击查看、修改或删除

**内容块分类：**

| 分类 | 标识色 | 描述 |
|------|--------|------|
| 自我介绍 | 档案红 | 核心身份描述 |
| 历史背景 | 藕紫 | 经历、学历、职业轨迹 |
| 能提供的 | 鼠尾草绿 | 资源、技能、人脉 |
| 具体需求 | 尘土橘 | 正在寻找什么 |
| 自定义 | 雾蓝 | 用户自己定义的分类 |

**交互细节：**
- 内容块卡片左侧带分类色条，右上角带 AI 生成标识（琥珀金角标）
- 支持文字输入新增；输入框 Placeholder 提示「试试用键盘语音输入，口述你的内容」
- 编辑态：点击内容块进入编辑，支持修改内容文字、分类、可见性
- 删除需二次确认，使用**中心弹窗（Modal）**确认（非 alert 弹窗）。单栏页面没有左侧面板，不应使用抽屉

**删除确认弹窗交互规格：**

| 交互项 | 规格 |
|--------|------|
| 弹窗类型 | 页面**中心弹窗**（Modal），复用设计系统 `.modal-content` |
| 背景遮罩 | 全屏半透明遮罩，`rgba(26, 26, 46, 0.45)`，点击遮罩取消 |
| 弹窗内容 | 删除确认文案 + 「确认删除」按钮（档案红 `.btn-danger`）+ 「取消」按钮（`.btn-secondary`） |
| 动效 | `modalIn`，`0.35s cubic-bezier(0.34, 1.56, 0.64, 1)`，与设计系统 `.modal-content` 一致 |
| **适用范围** | 单栏居中页面（个人主页 `/profile/[id]`）使用中心弹窗；左侧栏底部抽屉仅用于分栏页面（AI 采访页左侧面板、AI Lab 后台左侧面板） |

> **v1.3 变更说明**：删除确认交互从模糊的「底部抽屉」细化为「左侧栏底部升起抽屉」，含具体遮罩色值和动效参数。新增降级方案（中心弹窗）。

#### 故事四：内容变化自动更新标签和简介（P1）

> **作为** 一位校友，
> **我想要** 每次新增或修改内容块后，AI 以增量方式补充新的标签和简介段落，绝不覆盖我已经确认的内容，
> **以便** 我的主页始终反映最新的我，同时我永远不必担心辛苦写的内容被 AI 擅自改动。

**核心原则：用户是 Boss**
- AI 只做增量，不做覆盖
- 用户已确认的内容是「领地」，AI 不可侵入
- AI 的新增内容必须高辨识度展示，用户可一键抹除

**验收标准：**
- Given 用户已有一个内容块保存的个人主页
- When 用户新增或修改任意内容块
- Then AI 自动重新分析所有内容块，**以严格增量方式**补充新标签和简介增量段落，绝不修改或覆盖用户已确认的任何文字

**「用户是 Boss」增量机制：**

| 场景 | AI 行为 | UI 表现 |
|------|---------|---------|
| 简介来源为 `ai`（用户从未编辑过） | AI 可直接重写简介 | 简介区域打字机效果更新，Toast「内容已同步更新」 |
| 简介来源为 `user_edited`（用户修改或确认过） | AI **绝不修改**已有文字，仅在末尾追加增量段落 | 增量段落以**琥珀金浅色背景高亮**展示，旁边跟随 `[保留]` `[抹除]` 按钮 |
| 标签来源为 `ai`（用户从未编辑过） | AI 可直接更新标签列表 | 标签云动画刷新 |
| 标签来源为 `user_edited`（用户修改或确认过） | AI **绝不删除**已有标签，仅增量补充新标签 | 新标签以**琥珀金虚线框**展示，旁边跟随 `[×]` 抹除按钮 |

**增量段落的交互规则：**
- 增量段落以**琥珀金浅色背景高亮**（`rgba(196, 154, 108, 0.15)`），与用户已确认文字形成鲜明视觉区分
- 每个增量段落右侧跟随两个按钮：
  - `[保留]`：点击后，增量段落融入正文，高亮消失，来源标记为 `user_edited`
  - `[抹除]`：点击后，增量段落**瞬间消失，无需二次确认**——用户是 Boss，想删就删
- 增量标签以**琥珀金虚线框**展示，右上角带 `[×]` 按钮
  - 点击 `[×]`：标签**瞬间移除，无需二次确认**
  - 点击标签文字：保留标签，虚线框消失，转为正式标签，来源标记为 `user_edited`

**交互细节：**
- 内容块变更后，标签云区域显示 AI 解析骨架屏（琥珀金虚线边框 + 光泽 shimmer）
- 更新完成后显示 Toast：「AI 已补充新内容，请查看」（不用「保存成功」）
- 如 AI 处理耗时 > 3 秒，显示进度提示
- 增量段落/标签的保留或抹除操作实时生效，不弹确认框

**异常与中断恢复：**
- **AI 更新请求超时**（> 15s）：保留已有标签和简介不变，显示 Toast「更新较慢，请稍后刷新」
- **部分更新失败**（标签更新成功但简介失败）：已更新的标签保留，失败部分显示重试入口
- **网络中断**：前端缓存变更事件，网络恢复后自动重试，不丢失用户操作

> **v1.2 变更说明**：将 v1.1 的「建议模式」（查看/忽略弹窗）升级为「用户是 Boss」严格增量机制。核心变化：(1) AI 对 `user_edited` 内容绝不覆盖，只做增量追加；(2) 增量内容以琥珀金浅色背景高辨识度展示；(3) 一键抹除交互（`[保留]` `[抹除]`），无需二次确认。POC 阶段增量段落统一追加在简介末尾，不做"智能寻找合适位置嵌入"（工程复杂度过高）。

#### 故事五：手动控制图片生成（P2）

> **作为** 一位校友，
> **我想要** 手动触发个人展示图片的生成，而不是每次内容变化自动更新图片，
> **以便** 我能控制个人形象展示的版本，避免图片频繁变动带来的不一致感。

**验收标准：**
- Given 用户的标签和简介已更新
- When 系统提示「图片与当前内容可能不一致」并提供「更新图片」按钮
- Then 用户点击后触发 AI 生图服务生成新图片，旧图片保留历史记录
- And 生成的图片为**抽象意境图 / 隐喻视觉**，绝对不是写实人脸

**图片风格强约束：**

| 约束项 | 规则 | Prompt 控制方式 |
|--------|------|---------------|
| 禁止写实人脸 | 不得生成任何人物面部、人像、肖像 | Prompt 中强制包含 `no human face, no portrait, no figure` |
| 风格限定 | 抽象意境、几何排版、高级校刊插画、象征性静物 | Prompt 中限定 `abstract, metaphorical, editorial illustration, geometric composition` |
| 色彩基调 | 与设计系统一致 —— 莫兰迪色系、低饱和度、包浆感 | Prompt 中限定 `muted color palette, Morandi tones, vintage texture` |
| 比例 | 16:9 横版 | 尺寸参数由 AI 生图服务配置 |

**交互细节：**
- 图片区域 16:9 比例，圆角 6px
- 当标签/简介更新后图片未同步时，图片底部显示半透明提示条：「内容已更新，图片可能需要刷新」+ 琥珀金描边「更新图片」按钮
- 图片生成过程显示骨架屏 + 琥珀金 shimmer
- 旧图片保留可查看（历史记录入口）

**POC 降级说明：** 图片生成成本较高（API 费用 + 等待时间），POC 阶段仅做手动触发，不做自动生成。

> **v1.2 变更说明**：移除「DALL-E 3」和具体尺寸参数「1792×1024」，改为通用代称「AI 生图服务」，具体模型和参数由技术团队在火山引擎生态内选择。

#### 故事六：隐私信息的搜索可见与展示隐藏（P2）

> **作为** 一位校友，
> **我想要** 区分「直接展示在页面上」的信息和「仅通过 AI 搜索可见」的信息，
> **以便** 我能保护某些敏感或细节的隐私，同时不影响 AI 搜索匹配时被发现。

**验收标准：**
- Given 用户编辑某个内容块
- When 用户设置该内容块的可见性为「搜索可见但不展示」
- Then 该内容块的标签和关键词参与 AI 搜索匹配，但内容文字不直接显示在公开主页上，仅对用户自己可见
- And 该内容块提炼出的标签**不出现在公开主页的标签云中**，仅参与底层匹配逻辑

**可见性三级模型：**

| 可见性 | 标识色 | 主页展示 | 搜索匹配 | 标签云展示 | 他人可见 |
|--------|--------|---------|---------|-----------|---------|
| 公开 | 爬山虎绿 | ✅ | ✅ | ✅ | ✅ |
| 仅搜索可见 | 琥珀金 | ❌ | ✅ | ❌ | ❌ |
| 仅自己可见 | 占位灰 | ❌ | ❌ | ❌ | ❌ |

**标签可见性规则：**
- 标签继承其来源内容块的可见性设定
- `visibility=search_only` 的标签在公开主页标签云中隐藏
- 创建者本人可看到所有标签（含 search_only），以虚线框 + 琥珀金标识区分
- 标签仅在底层搜索/匹配逻辑中参与计算，不暴露给浏览者

**交互细节：**
- 内容块底部显示可见性标签（图标 + 文字）
- 点击可见性标签弹出底部抽屉，三选一
- 「仅搜索可见」的内容块在个人主页上以虚线轮廓 + 半透明文字显示，仅创建者可见
- 切换可见性时，标签云实时刷新（公开 → search_only：标签从云中消失；反之出现）

**POC 降级说明：** 隐私是长期命题，POC 只实现三级切换，不做复杂权限管理。

#### 故事七：直接修改 AI 生成内容（P1）

> **作为** 一位校友，
> **我想要** 直接修改 AI 生成的标签和文字描述，
> **以便** 确保展示内容准确反映我的真实意图，防止 AI 误解或过度推断。

**验收标准：**
- Given 用户查看 AI 生成的标签或简介
- When 用户点击「编辑」并修改标签或文字
- Then 系统保存修改，将来源标记为 `user_edited`，后续 AI 自动更新时**绝不覆盖**此内容，仅以增量方式补充（见故事四）

**来源优先级规则（「用户是 Boss」原则）：**

| 内容 | `source` 值 | AI 更新时行为 |
|------|------------|-------------|
| AI 生成、用户未修改 | `ai` | 可被 AI 更新 |
| 用户手动修改过 | `user_edited` | **绝对不被 AI 修改或覆盖**，AI 仅做增量补充（见故事四增量机制） |
| 用户手动新增 | `user_edited` | **绝对不被 AI 修改或覆盖** |

**交互细节：**
- 标签云：点击标签进入编辑态，可修改文字、删除标签、手动新增标签
- 简介文字：点击进入编辑态，支持富文本编辑
- 用户修改过的内容标记来源为 `user_edited`，与 `ai` 来源区分
- `user_edited` 标签/简介在 UI 上以微弱视觉标识区分（如更深的边框色），让用户知道此内容「受保护」
- 保存后显示 Toast：「已保存，AI 不会修改您的内容」
- 修改记录写入 AI 处理日志，供产品经理分析偏差模式

> **v1.2 变更说明**：将「AI 仅提供建议」强化为「AI 绝不覆盖，仅做增量补充」，与故事四的「用户是 Boss」增量机制统一。Toast 文案从「AI 将在下次更新时参考您的修改」改为「AI 不会修改您的内容」——更直接、更有掌控感。

## 4. 设计方向

### 4.1 设计 DNA

第二代产品（AI 自我介绍）是第一代产品（基地）的聚焦演进，设计 DNA 必须一脉相承。这不是一个新 App，是基地的新能力。

**核心关键词**：克制、体面、包浆感、高级校刊、数字纪念册

**品牌参考**：Apple（极简留白）× Stripe（编辑式排版品质）× Airbnb（温暖人文底色）

**视觉基调**：
- 参考《Monocle》与《Kinfolk》的排版比例 —— 大留白、重字体层级、图片与文字的诗意穿插
- 色彩带有「包浆感」 —— 所有颜色经过灰度调和，呈现莫兰迪色系的褪色质感
- 以微弱悬浮阴影表现卡片质感，禁止弥散阴影

### 4.2 设计系统继承

| 继承维度 | 第一代「基地」 | POC 继承方式 |
|---------|--------------|-------------|
| 视觉基调 | 克制、体面、包浆感 | **完全继承** — CSS Token 直接复用 |
| 色彩体系 | 莫兰迪色系、档案红、旧黑板青、琥珀金 | **完全继承** — 新增 AI 场景色彩映射 |
| 排版规则 | 衬线标题 + 无衬线正文、大留白 | **完全继承** — 字体 Token 复用 |
| 组件风格 | 微圆角（4-6px）、微阴影、物理阻尼动画 | **完全继承** |
| AI 视觉语言 | 意图舱 4 态、琥珀金解析高亮、骨架屏 | **继承 + 扩展** — 核心场景 |
| 标签系统 | 4 色莫兰迪（属于/关注/提供/需要） | **继承 + 简化** — 4 色直接复用 |
| 空状态 | 毛玻璃 + 占位文案 | **继承** — 复用设计系统空状态 Token |

### 4.4 设计禁忌

| # | 禁忌 | 说明 |
|---|------|------|
| 1 | 禁止高饱和度原色 | AI 高亮用琥珀金，不用荧光色 |
| 2 | 禁止活泼渐变和卡通插画 | AI 生成图片也需符合「数字纪念册」基调，且必须为抽象意境风格 |
| 3 | 禁止大圆角（> 8px） | 保持 4-6px |
| 4 | 禁止生硬弹窗 | 所有过渡带物理阻尼感；AI 增量内容用高亮 + 一键按钮，不用弹窗确认 |
| 5 | 禁止通用互联网文案 | 不用「提交」「保存成功」，用「已记录」「内容已同步更新」 |
| 6 | 禁止对比度未达标的文字 | AI 标签必须通过 WCAG AA 对比度验证 |
| 7 | 禁止生成写实人脸图片 | AI 生成图片必须为抽象意境 / 隐喻视觉，避免恐怖谷效应 |
| 8 | 禁止 AI 覆盖用户已确认的内容 | 用户是 Boss —— AI 只做增量，不做覆盖 |

> **v1.2 变更说明**：禁忌 #4 强化——增量内容用高亮+一键按钮而非弹窗；禁忌 #8 新增——将「用户是 Boss」原则上升为设计禁忌。

### 5.2 关键页面规格

#### A. 创建方式选择页（/create）

**布局**：居中两列卡片选择
- 左卡：快速创建 —— 文字输入图标 + 描述文案「口述或键入自我介绍，AI 一键生成主页」
- 右卡：AI 采访模式 —— 对话图标 + 描述文案「跟 AI 聊天，逐步完善你的主页」

#### B. 快速创建页（/create/quick）

**布局**：上方大区域输入 + 下方实时预览
- 顶部：AI 意图输入舱（4 态视觉状态机）
  - Idle 态：Placeholder 呼吸闪烁，提示「试试用键盘语音输入，口述你的自我介绍」
  - Parsing 态：琥珀金跑马灯边框

  - Error 态：档案红边框 + 微震动
- 大文本输入框：多行，支持长文本，旁附语音输入引导图标（非录音按钮）

  - 输入框撑高时，下方预览区域自然收缩，但保持最小可读高度（200px）
  - 用户始终可以在完整页面中上下滚动查看所有内容
- 下方：标签云 + 内容块 + 简介实时预览区域

> **v1.3 变更说明**：新增大文本输入框自动撑高规格，明确禁止内部滚动，下方预览区域保持最小可读高度。

#### C. AI 采访页（/create/interview）

**布局**：左右分栏
- 左侧（60%）：对话面板
  - AI 提问气泡（白色底 + 分类色标）
  - 用户回答气泡（档案红淡底）
  - 底部：文字输入框 + 发送按钮
  - **顶部常驻**：「结束采访，生成我的主页」按钮 + 维度覆盖进度（如「已覆盖 2/4 维度」）
- 右侧（40%）：实时预览
  - 标签云
  - 内容块列表（实时新增）
  - 简介文字（打字机效果更新）

#### D. 个人主页页（/profile/[id]）

**布局**：人物志风格单栏布局
- 顶部：个人信息卡（姓名 + 班级 + AI 生成图片）
  - 图片 16:9 比例，抽象意境风格
  - 图片过期时底部半透明提示条 + 琥珀金「更新图片」按钮
- 中部：简介区域
  - 用户已确认文字：正常展示
  - **AI 增量段落**：琥珀金浅色背景高亮，右侧 `[保留]` `[抹除]` 按钮
  - 可点击编辑（编辑后来源标记为 `user_edited`）
- 中部：标签云（4 色莫兰迪）
  - `search_only` 标签仅创建者可见，以虚线框 + 琥珀金标识
  - **AI 增量标签**：琥珀金虚线框 + `[×]` 抹除按钮
- 底部：内容块列表
  - 每块：左侧分类色条 + 内容 + AI 角标 + 可见性标识
  - 支持新增 / 编辑 / 删除操作
  - **删除确认**：点击删除后，在左侧内容块面板底部升起确认抽屉（含确认/取消按钮 + 遮罩）；降级方案为中心弹窗
- 右下角浮动：「新增内容块」按钮

**空状态定义（Empty State）：**

> 当用户刚进入个人主页，没有任何内容块和标签时，显示空状态占位。参考第一代设计系统（`DESIGN-第一代产品参考设计.md`）的空状态规范。

| 区域 | 空状态 UI | 视觉规格 |
|------|----------|---------|
| 简介区域 | 衬线体占位文案 + 半透明装饰线 | 占位文案：「你的故事，将由 AI 与你一起书写」，字色 `#7F8C8D`（占位灰），字体 `var(--font-serif)`，字号 16px。上下各一条 `rgba(196,154,108,0.2)` 水平装饰线 |

#### E. AI Lab 后台（/admin）

## 6. 数据模型

### 6.1 核心实体

```
User（用户）
  ├── id, name, className, avatar, createdAt
  └── Profile 1:1

Profile（个人主页）
  ├── id, userId
  ├── userConfirmedIntro  String?   // 用户已确认的简介（user_edited 或已采纳的 AI 内容）
  ├── aiDeltaIntro        String?   // AI 增量简介段落（待用户保留/抹除）
  ├── introSource         String    @default("ai")  // "ai" | "user_edited"
  ├── imageUrl, imageHistory, imageOutdated
  ├── createdAt, updatedAt
  ├── ContentBlock 1:N
  └── ProfileTag 1:N

ContentBlock（内容块）
  ├── id, profileId, category, title, content
  ├── visibility（public | search_only | private）
  ├── source（user | ai_extracted | ai_interview）
  └── order, createdAt, updatedAt

ProfileTag（标签）
  ├── id, profileId, tagText, tagType
  ├── tagType（belong | follow | offer | need）
  ├── source（ai | user_edited）
  ├── visibility String  @default("public")  // 继承来源内容块的可见性：public | search_only | private
  ├── isDelta    Boolean @default(false)     // 是否为 AI 增量标签（待用户保留/抹除）
  └── order

AILog（AI 处理日志）
  ├── id, userId, action, input, output
  ├── model, promptUsed, duration
  ├── userModified, modification
  ├── modificationType String?  // "substantive"（实质性纠偏）| "cosmetic"（轻微润色）| "delta_rejected"（增量抹除）| null（未修改）
  └── createdAt

PromptVersion（Prompt 版本）
  ├── id, promptKey, content, version
  ├── isActive, createdAt
  └── promptKey（extract_tags | generate_intro | generate_image | interview_guide）

PromptTemplate（Prompt 配置模板）
  ├── id, name, description, config(String)
  ├── isPublic Boolean @default(true)
  └── createdAt
  └── config 存储完整 JSON：{ model, temperature, targetDimensions, maxRounds, prompts: { extract_tags, generate_intro, generate_image, interview_guide } }
```

> **v1.5 变更说明**：
> - 新增 `PromptTemplate` 实体，用于保存/加载完整的 Prompt 配置模板
> - `extract-tags` API 新增 `existingContext` 可选参数，支持增量更新场景传入已有档案上下文

> **v1.2 变更说明**：
> - `Profile`：将 `aiSuggestedIntro` 改为 `aiDeltaIntro`（反映「增量」而非「建议」的语义），`userConfirmedIntro` 保留
> - `ProfileTag`：将 `isSuggested` 改为 `isDelta`（与 `aiDeltaIntro` 命名一致）
> - `AILog.modificationType`：新增 `"delta_rejected"` 值，用于统计增量抹除率

### 6.2 内容块分类枚举

| category 值 | 中文名 | 对应标签类型 | 标识色 |
|------------|--------|------------|--------|
| `self_intro` | 自我介绍 | belong | 档案红 |

## 7. 技术约束与选型方向

> **原则**：POC 技术选型优先「零运维、快搭建、易迭代」，不追求生产级架构。

| 决策项 | 选型方向 | 理由 |
|--------|---------|------|
| 前端框架 | Next.js 14 (App Router) + TypeScript | 全栈能力、零配置、最快原型搭建 |
| UI 层 | Tailwind CSS + shadcn/ui | 组件可控、与设计系统 Token 融合方便 |
| 输入方式 | 纯文字输入 + 引导系统键盘语音转文字 | 零语音工程成本、零兼容性风险、聚焦 AI 能力验证 |
| AI 基础设施 | **火山引擎（Volcano Engine）** | 文本生成（LLM）及其他 AI 能力统一接入火山引擎，具体模型版本由技术团队评估选择 |
| AI 生图服务 | 火山引擎生态内的 AI 生图能力 | 手动触发，控制成本；Prompt 强控风格；具体模型由技术团队评估 |
| 后端 | Next.js API Routes | 前后端一体，减少部署复杂度 |
| 数据存储 | SQLite + Prisma ORM | 零运维、本地文件即数据库 |
| 配置管理 | **数据库表 PromptTemplate** 存储 Prompt 配置模板 | 支持云端保存/加载/删除 + 本地 JSON 导入导出 |
| 部署 | 云端服务器 | 用户已准备好，部署 Web 版供朋友测试 |
| 设备适配 | **桌面端专属** | POC 仅面向桌面端（≥ 1024px 宽度），移动端访问显示引导页，不做响应式布局 |

> **v1.3 变更说明**：新增「设备适配」技术决策项——桌面端专属，移动端引导页重定向，不做响应式布局。

## 9. 指标体系

### 9.1 北极星指标

**个人主页创建完成率** = 成功创建个人主页的用户数 / 开始创建流程的用户数

POC 目标：≥ 70%

### 9.2 驱动指标

| 指标 | 定义 | 目标 | 数据来源 |
|------|------|------|---------|
| 口述路径完成率 | 快速创建流程完成数 / 快速创建启动数 | ≥ 70% | 前端埋点 |
| AI 新增标签占比 | 用户未主动提及的标签数 / 总标签数 | ≥ 30% | AI Log 分析 |
| 实质性纠偏率 | 用户对 AI 核心语义的实质性纠偏次数 / AI 生成总次数 | ≤ 30% | AI Log 的 modificationType="substantive" |
| AI 增量抹除率 | 用户对 AI 增量内容点击抹除/`[×]`的次数 / AI 增量内容总数 | ≤ 40% | AI Log 的 modificationType="delta_rejected" |
| Prompt 迭代速度 | 从修改 Prompt 到验证效果的时长 | < 5 分钟 | 操作计时 |

> **v1.2 变更说明**：新增「AI 增量抹除率」驱动指标，用于衡量 AI 增量内容的质量。抹除率高说明增量不准，需调优 Prompt。

### 9.3 健康指标

| 指标 | 定义 | 告警阈值 |
|------|------|---------|
| AI 标签提取耗时 | 从提交到返回标签的时长 | > 5s |
| 图片生成耗时 | 从触发到返回图片的时长 | > 30s |
| SSE 连接中断率 | 采访模式 SSE 连接异常中断的比例 | > 10% |
| 页面加载时长 | 首屏可交互时间（桌面端） | > 3s |

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|---------|
| AI 标签提取不够准确 | 中 | 中 | 后台 Prompt 编辑器快速迭代；收集日志分析偏差模式 |
| AI 生图服务生成写实人脸 | 低 | 高 | Prompt 硬编码风格约束前缀（不可删除）；人工审核首批生成结果 |
| 图片生成慢 / 贵 | 高 | 低 | 默认不自动生成（手动触发）；降级用占位图 |
| SSE 流式输出中断 | 中 | 中 | 前端自动重试（3 次）+ 已接收内容保留 + 断点续传 |
| AI API 超时/安全拦截 | 低 | 中 | 超时提示 + 重试入口；安全拦截友好提示；部分更新保留 |
| 测试朋友数量不足 | 低 | 高 | 提前确认 3-5 位测试人；自己扮演多位用户 |
| 云端服务器部署问题 | 低 | 中 | 确保服务器环境（Node.js + 端口开放）提前验证 |
| 用户不理解系统键盘语音输入 | 中 | 低 | 输入框 Placeholder 引导 + 首次使用弹出简短说明 |
| 用户对 AI 增量内容产生"信息疲劳" | 中 | 中 | 增量内容高辨识度展示 + 一键抹除，让用户感觉可控；如抹除率 > 60% 则优化增量 Prompt |
| 移动端用户误入 POC 页面 | 中 | 低 | UserAgent 检测自动重定向至 `/mobile` 引导页；引导页提供可复制的桌面端 URL |

> **v1.3 变更说明**：新增「移动端用户误入 POC 页面」风险项，应对策略为 UserAgent 检测 + 引导页重定向。

## 11. 迭代判断标准

| 观察指标 | 如果 | 则 |
|---------|------|-----|
| 口述路径完成率 < 50% | 用户放弃口述路径 | 优化输入框引导文案，或简化创建流程 |
| AI 新增标签 < 15% | AI 采访没挖出信息 | 优化采访 Prompt + 追问逻辑 |
| 实质性纠偏率 > 50% | AI 生成核心信息质量不够 | 分析日志中高频纠偏模式 → 针对性调 Prompt |
| AI 增量抹除率 > 60% | AI 增量内容不准或不需要 | 优化简介生成 Prompt，减少不必要的增量输出；或降低增量触发频率 |
| 产品经理改 Prompt 后效果更差 | 迭代方向有误 | 增加两版 Prompt 对比功能 |
| 口述路径使用率 > 80% | 用户高度认可口述创建 | 口述创建作为下一代核心体验 |
| 采访平均轮数 < 3 轮 | 用户过早退出采访 | 优化采访开场体验，降低认知门槛 |
| SSE 中断率 > 10% | 流式体验不稳定 | 增强重试机制，或降级为非流式请求 |

> **v1.2 变更说明**：新增「AI 增量抹除率 > 60%」迭代判断标准。

> **核心原则**：把「写个人主页」这个痛苦的事，变成「跟 AI 聊天」这个自然的事。
> **用户是 Boss**：AI 只做增量，不做覆盖。用户已确认的内容是领地，AI 不可侵入。
> **桌面端专属**：POC 只做桌面端体验，移动端用户引导至桌面设备访问。
> POC 的目标不是做完美产品，而是验证：**AI 能不能让校友们真的愿意创建并维护自己的个人主页**。
