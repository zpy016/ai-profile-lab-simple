## 二、Prompt（提示词）是怎么被使用的？

### 2.1 大模型调用的基本格式

所有文字 AI 调用（工作流1、工作流2）都使用 **OpenAI Chat Completions API** 格式：

```json
{
  "model": "ep-20260609201117-gcqzt",
  "messages": [
    { "role": "system", "content": "【这里放右侧 Prompt 编辑器里的内容】" },
    { "role": "user", "content": "【这里放用户输入的文字】" }
  ],
  "temperature": 0.7,
  "response_format": { "type": "json_object" }
}
```

**核心规则**：
- `system` 角色 = 右侧 Prompt 编辑器里的内容（你的调优指令）
- `user` 角色 = 用户实际输入的文字
- 两者拼接后，通过 OpenAI SDK 发送到火山方舟
- 火山方舟返回 AI 的回复，后端解析后返回给前端展示

### 2.2 工作流1：快速录入的完整流程

```
用户在前端输入框打字/语音输入
        │
        ▼
用户点击「生成主页」按钮
        │
        ▼
前端把以下内容打包成 JSON，POST 到 /api/chat：
  ├─ systemPrompt: 右侧「标签提取 & 内容分块 Prompt」框里的文字
  ├─ userPrompt: 用户在输入框里写的自我介绍
  ├─ model: 右侧「模型选择器」选中的 endpoint ID
  ├─ temperature: 右侧滑块的值
  └─ responseFormat: { type: "json_object" }  ← 要求 AI 返回 JSON
        │
        ▼
后端 /api/chat/route.ts 接收请求：
  ├─ 读取 .env.local 里的 ARK_API_KEY
  ├─ 用 OpenAI SDK 创建客户端（baseURL=火山方舟）
  ├─ 调用 chat.completions.create({...})
  └─ 等待 AI 返回
        │
        ▼
火山方舟大模型接收到请求后：
  ├─ 把 systemPrompt 当作"系统指令"
  ├─ 把 userPrompt 当作"用户输入"
  └─ 按指令分析内容，生成 JSON 格式的输出
        │
        ▼
后端把 AI 返回的 JSON 字符串原封不动传回前端
        │
        ▼
前端解析 JSON：
  ├─ blocks[] → 文本块列表（分类色条卡片）
  ├─ tags[] → 标签云
  └─ intro → 简介段落
        │
        ▼
左侧展示结果，用户可编辑/删除文本块和标签
        │
        ▼
用户点击「确认并发布到个人主页」
        │
        ▼
数据写入 localStorage，自动跳转到「个人主页」Tab 预览
```

### 2.3 工作流2：AI 采访的完整流程

```
用户切换到「AI 采访」Tab
        │
        ▼
页面自动发起第一次请求（POST /api/chat）：
  ├─ systemPrompt: 右侧「采访引导 Prompt」
  ├─ userPrompt: "开始采访"
  └─ 其他参数同上
        │
        ▼
AI 返回第一个问题（如"请先简单介绍一下自己吧"）
        │
        ▼
前端把问题显示在对话面板（左侧气泡）
        │
        ▼
用户输入回答，点击发送
        │
        ▼
前端再次 POST /api/chat，但这次：
  ├─ systemPrompt = 采访引导 Prompt + 完整的对话历史
  ├─ userPrompt = 用户刚刚输入的回答
  └─ 要求 AI 返回 JSON：{ nextQuestion, hiddenBlocks, extractedTags, introSnippet }
        │
        ▼
后端转发给火山方舟大模型
        │
        ▼
AI 返回：
  ├─ nextQuestion → 显示在下一条 AI 气泡里
  ├─ hiddenBlocks → 存入「仅搜索可见」的特殊文本块
  ├─ extractedTags → 显示在标签云
  └─ introSnippet → 累计为简介素材
        │
        ▼
重复上述对话过程，直到：
  · 轮数达到上限（默认 6 轮）
  · 或用户点击「结束采访」
        │
        ▼
结束采访时，再发一次请求：
  ├─ systemPrompt = 标签提取 Prompt + hidden 文本块上下文
  ├─ userPrompt = 完整对话记录
  └─ 要求生成最终的 blocks + tags + intro
        │
        ▼
生成最终结果，用户确认后发布到 Profile
```

### 2.4 图片生成的完整流程

```
用户在「个人主页」Tab 看到占位图
        │
        ▼
点击「生成概念图」按钮
        │
        ▼
前端把以下内容打包，POST 到 /api/image：
  ├─ prompt = 右侧「图片生成 Prompt」的内容
  │           + 用户简介
  │           + 用户标签列表
  └─ model = 右侧「生图模型 Endpoint」
        │
        ▼
后端 /api/image/route.ts 接收请求：
  ├─ 读取 ARK_API_KEY
  ├─ 直接调用 fetch() 到火山方舟 images/generations 端点
  │   （注意：生图不走 OpenAI SDK，走原生 HTTP）
  └─ 请求体：
      {
        "model": "ep-20260609212907-sw6cc",
        "prompt": "...",
        "size": "2K",
        "output_format": "png",
        "watermark": false,
        "response_format": "url"
      }
        │
        ▼
火山方舟 Seedream 模型生成图片
        │
        ▼
返回图片 URL（24小时有效）
        │
        ▼
前端用 <img> 标签直接展示图片
```

## 三、右侧面板调优工具详解

### 3.1 你修改 Prompt 后会发生什么？

右侧面板有 4 个 Prompt 编辑器。你修改其中任何一个后：

1. **修改立即生效**（不需要重启服务器）
2. 下一次用户点击「生成」或「发送」时，前端会使用**最新的 Prompt 内容**
3. AI 的行为会相应改变

### 3.2 Temperature 的作用

| 值 | 效果 |
|----|------|
| 0.0 | AI 输出非常稳定、保守、可预测 |
| 0.7（默认）| 平衡，既有创意又有稳定性 |
| 1.0 | AI 输出更发散、更有创意、但也更不可控 |

### 3.3 模型切换

下拉框选择不同模型后，下一次请求就会使用新模型。不需要刷新页面。

## 四、Hidden 文本块的特殊机制

这是工作流2的核心创新点：

**什么是 Hidden 文本块？**
- 用户在采访中透露的、不愿意直接展示在主页上的信息
- 例如："其实我正在考虑换工作，但不想让现同事知道"
- 这些信息标记为 `category: "hidden"`

**Hidden 文本块的使用场景：**
- 不在「个人主页」预览中显示
- 但会在右侧面板的「Hidden 文本块」区域列出
- 当你点击「模拟其他同学提问」时，系统会把所有 hidden 内容作为上下文传给 AI
- AI 在回答问题时，可以引用这些隐藏信息（验证"搜索可见"的能力）

**数据流向：**
```
Hidden 文本块 → localStorage 保存
                     │
                     ▼
         模拟提问时作为 system prompt 的一部分
                     │
                     ▼
         AI 回答问题时可以引用这些信息
```

## 五、数据持久化

所有数据都存在浏览器的 **localStorage** 中：

| 数据 | 存储位置 |
|------|---------|
| 用户输入的文字 | localStorage |
| AI 生成的文本块/标签/简介 | localStorage |
| 对话历史 | localStorage |
| Hidden 文本块 | localStorage |
| 右侧面板的 Prompt 配置 | localStorage |
| 图片 URL | localStorage |

**特点**：
- 刷新页面数据不丢
- 换浏览器/换电脑数据不共享
- 清理浏览器缓存会丢失数据

## 七、常见问题

**Q: 为什么 AI 返回的结果不是 JSON？**
A: 可能是 Prompt 不够明确，或者模型不支持 `json_object` 格式。尝试在 Prompt 里加一句"输出严格 JSON 格式"。

**Q: 生图失败了怎么办？**
A: 检查右侧面板的「生图模型 Endpoint」是否正确。默认是 `ep-20260609212907-sw6cc`。也可以尝试直接用 Model ID `doubao-seedream-5-0-260128`。

**Q: 我能同时修改多个 Prompt 吗？**
A: 可以。所有 Prompt 的修改都是实时保存到 localStorage 的，互不影响。

**Q: 图片 URL 为什么 24 小时后失效？**
A: 这是火山方舟的限制。生成的图片 URL 只保留 24 小时。如需长期保存，需要另外下载存储。
