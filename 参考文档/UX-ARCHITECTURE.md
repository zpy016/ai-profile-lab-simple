## 1. 当前架构诊断：10 个用户体验断点

> **v1.2 变更**：(1) Admin 后台 UX 全面升级：左侧从只读预览改为三场景可交互沙盒控制台；(2) 新增模板管理 UX 流程（保存/加载/导出/导入）；(3) 新增 extract-tags API `existingContext` 参数的 UX 说明（增量更新场景）；(4) 更新 Admin 后台信息架构和页面规格
>
> **v1.1 变更**：修复 10 个用户体验断点，定义完整用户旅程

经过对 PRD（v1.5）、DESIGN.md（v1.3）和 6 个 mockup 页面的系统审阅，识别出以下 10 个用户体验断点：

| # | 断点 | 严重度 | 影响用户旅程 | 现状 |
|---|------|--------|-------------|------|
| BP1 | 首页 `/` 未定义 | 🔴 严重 | 新用户没有入口点 | ✅ 已实现：品牌展示 + 状态路由 + CTA |
| BP2 | 创建→主页过渡缺失 | 🔴 严重 | 完成创建后用户"悬空" | ✅ 已实现：Review + Published 庆祝状态 |
| BP3 | AI Lab 导航入口缺失 | 🟡 中等 | 产品经理无法从用户界面进入后台 | ✅ 已实现：TopNav 齿轮 + Landing 底部链接 |
| BP4 | 内容块新增/编辑流程未定义 | 🔴 严重 | 用户无法理解如何添加和编辑内容 | ✅ 已实现：inline add + edit-in-place |
| BP5 | 快速创建与采访模式无法切换 | 🟡 中等 | 用户选错路径后无法挽回 | ✅ 已实现：降级切换 + preview 数据传递 |
| BP6 | 无草稿自动保存 | 🟡 中等 | 用户中途离开丢失所有进度 | ✅ 已实现：quick + interview 双 localStorage 草稿 |
| BP7 | SSE 断线恢复后采访上下文不清 | 🟡 中等 | 网络中断后用户困惑 | ✅ 已实现：3 次自动重试 + 手动恢复 + "稍后继续" |
| BP8 | 新增内容块触发流程未定义 | 🔴 严重 | 浮动按钮点击后发生什么？ | ✅ 已实现：inline 展开 → 分类 pill → 确认 |
| BP9 | 个人主页状态机未定义 | 🟡 中等 | empty/draft/active/dirty 状态无视觉区分 | ✅ 已实现：4 态全区分 + 提示条 |
| BP10 | 创建完成后的 CTA 缺失 | 🟡 中等 | 用户不知道"下一步做什么" | ✅ 已实现：3 个 CTA（查看/完善/返回） |

## 2. 优化后的用户旅程架构

### 2.1 Profile 生命周期状态机

```
Visitor ──login──▸ Empty Profile ──start──▸ Creating ──submit──▸ Active Profile
                     ▲                       │                        │
                     │                       │                        │
                     │              ┌────────┴────────┐         edit/add delta
                     │              │                 │               │
                     │          Quick Create    AI Interview           │
                     │              │                 │               │
                     │              └────┬────────────┘               │
                     │                   ▼                            │
                     │            Creation Review                     │
                     │            (accept/reject)                     │
                     │                   │                            │
                     │            Auto-save Draft ────confirm──▸      │
                     │                                              │
                     ◄─────────────── Profile Dirty ◄───────────────┘
                                      (has unconfirmed
                                       AI delta content)
```

**状态定义：**

| 状态 | 数据条件 | 页面行为 | 导航逻辑 |
|------|---------|---------|---------|
| `empty` | Profile 不存在 | 显示空状态个人主页 + 引导 CTA | 自动跳转 `/create` |
| `draft` | Profile 存在但 `publishedAt = null` | 显示创建预览 + 编辑/提交按钮 | 停留在创建页面 |
| `active` | Profile 已发布，无未确认 delta | 正常展示个人主页 | 停留在 `/profile/[id]` |
| `dirty` | Profile 有未确认的 AI 增量内容 | 增量内容高亮展示 + 操作按钮 | 停留在 `/profile/[id]` |

### 2.2 核心用户旅程（3 条路径）

#### 旅程 A：快速创建 → 发布主页

```
/ (Landing)
  │
  ▼
/create (选择创建方式)
  │
  ▼
/create/quick (口述/键入自我介绍)
  │
  ├── 输入文字 → AI 意图舱 Parsing → Field_Filling → 预览生成
  │
  ├── [新增] Creation Review 步骤
  │     ├── 标签云：逐个 Accept / Reject
  │     ├── 简介：Accept / Edit / Reject delta
  │     ├── 内容块：逐个查看，支持编辑
  │     └── 图片：Accept 默认占位 / 手动触发生成
  │
  ├── 点击「确认并发布」
  │
  └── [新增] Published 庆祝状态
        ├── Toast「你的主页已上线」
        ├── CTA 1：「查看我的主页」→ /profile/[id]
        ├── CTA 2：「继续完善」→ /profile/[id] 编辑态
        └── CTA 3：「返回基地」→ /
```

#### 旅程 B：AI 采访 → 发布主页

```
/ (Landing)
  │
  ▼
/create (选择创建方式)
  │
  ▼
/create/interview (AI 对话采访)
  │
  ├── AI 逐维度提问 → 用户回答 → 右侧实时预览
  │
  ├── 收敛条件满足（4维度 / 10轮 / 用户主动结束）
  │
  ├── [新增] Interview Summary 步骤
  │     ├── AI 生成采访总结 + 完整标签 + 简介建议
  │     ├── 用户确认/编辑总结
  │     └── 生成预览
  │
  └── 同旅程 A 的 Published 庆祝状态
```

#### 旅程 C：查看/编辑已有主页

```
/profile/[id] (个人主页)
  │
  ├── [新增] 入口识别
  │     ├── 创建者本人：显示编辑模式入口 + AI 增量提示
  │     └── 浏览者：只读展示（POC 阶段同一批人切换角色）
  │
  ├── 编辑简介 → 来源标记 user_edited → AI 不再覆盖
  │
  ├── 新增内容块 → [新增] 内联创建流程
  │     ├── 点击浮动按钮「+ 新增内容块」
  │     ├── 展开 inline 输入区域（非弹窗，非跳转）
  │     ├── 输入内容 → 选择分类 → 确认
  │     ├── AI 自动增量更新标签和简介
  │     └── 增量内容以琥珀金高亮展示
  │
  └── 编辑/删除内容块 → AI 增量更新 → delta 确认
```

### 3.2 Creation Review 步骤 — BP2, BP10 修复

> **当前问题**：创建完成后无"完成"状态，用户直接悬空。

**触发条件**：快速创建或 AI 采访完成后，进入个人主页之前。

**交互位置**：在创建页面原位展示，不跳转新路由。内容区域淡出替换为 Review 视图。

```
┌─ 原创建页内容区 ──────────────────────────────────────────┐

│  ✅ 「AI 已完成分析」(旧黑板青, 16px)                       │
│                                                            │
│  ─── 分割线 ───                                            │
│                                                            │
│  ┌─ 标签云 ──────────────────────────────────────────┐     │
│  │  [实验中学 2006] [清华计算机系] [AI 创业]            │     │
│  │  [增量标签: 寻找技术合伙人 ✕] [增量标签: 海淀咖啡 ✕]  │     │
│  │  提示：「琥珀金标签是 AI 新增的，点击确认或移除」      │     │
│  └────────────────────────────────────────────────────┘     │
│                                                            │
│  ┌─ 简介 ────────────────────────────────────────────┐     │
│  │  前大厂产品负责人，现 AI 创业者...（用户已确认文字）    │     │
│  │  ┌─ AI 增量段落 (琥珀金浅底) ─────────────────┐    │     │
│  │  │  目前在寻找技术合伙人...     [保留] [抹除]    │    │     │
│  │  └────────────────────────────────────────────┘    │     │
│  └────────────────────────────────────────────────────┘     │
│                                                            │
│  ┌─ 内容块预览 ──────────────────────────────────────┐     │
```

**点击「确认并发布」后：**

2. 页面平滑过渡至 Published 庆祝状态（见 3.3）
3. 触发 AI 图片生成（异步，后台执行，不阻塞用户）

```
│  │  [查看我的主页]  → /profile/[id]   (primary btn)     │      │
```

### 3.4 新增内容块内联流程 — BP4, BP8 修复

> **当前问题**：浮动按钮点击后无后续规格，编辑流程缺失。

**交互方式**：个人主页页面内 inline 展开，不跳转新页面，不弹全屏弹窗。

```
点击浮动按钮「+ 新增内容块」
  │
  ▼
个人主页底部内容块列表下方展开输入区域：

┌─ 新增内容块输入区 ──────────────────────────────────────┐
│                                                          │
│  分类选择：[自我介绍] [历史背景] [能提供的] [具体需求] [自定义] │
│  （pill 切换，选中态档案红底 + 白字）                       │
│                                                          │
│  ┌─ 输入框 ───────────────────────────────────────┐      │
│  │  试试用键盘语音输入，口述你想添加的内容...           │      │
│  │  (自动撑高, overflow:hidden, 同 /create/quick)     │      │
│  └──────────────────────────────────────────────────┘      │
│                                                          │
│  [取消] (ghost)            [添加] (primary)               │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

**点击「添加」后：**

1. 内容块即时出现在列表中（来源标记 `user`）
2. AI 后台增量更新标签和简介（异步，不阻塞）
3. 增量内容以琥珀金高亮出现在标签云和简介区域
4. Toast：「已记录，AI 正在补充相关内容」
5. 输入区域收起，浮动按钮恢复

**编辑已有内容块：**

1. 点击内容块的「编辑」按钮
2. 内容块原地切换为编辑态（文字变 textarea，分类变 pill 选择器）
3. 编辑完成点击「保存」→ 来源标记为 `user_edited`
4. Toast：「已保存，AI 不会修改您的内容」
5. AI 增量更新标签/简介（基于修改后的内容重新分析）

### 3.5 路径切换机制 — BP5 修复

> **当前问题**：快速创建和采访模式完全隔离，用户无法切换。

**策略**：允许从采访模式降级到快速创建，不允许反向。

```
/create/interview 顶部常驻栏新增「切换到快速创建」按钮
  │
  ▼
点击后弹出确认弹窗（Modal）：
  「当前采访进度将被保留为草稿，你可以随时回来继续」
  [继续采访]  [切换到快速创建]
  │
  ├── 继续采访 → 关闭弹窗，继续对话
  │
  └── 切换到快速创建 →
        ├── 已有的内容块和标签自动带入 /create/quick 预览区
        ├── 输入框可继续补充内容
        └── AI 基于已有内容增量更新
```

**设计理由**：
- 从采访切换到快速创建是"降级"（已有内容可以带走）
- 从快速创建切换到采访是"升级"（需要 AI 理解已有内容后开始对话，逻辑复杂）
- POC 阶段只做降级切换，不做升级切换

### 3.6 草稿自动保存 — BP6 修复

> **当前问题**：用户中途关闭页面，所有输入丢失。

**策略**：前端 localStorage + 服务端双重保存。

| 保存时机 | 保存位置 | 保存内容 |
|---------|---------|---------|
| 每次内容块生成 | 服务端 DB | ContentBlock + ProfileTag + Profile 字段 |
| 每次标签确认/抹除 | 服务端 DB | ProfileTag 状态 + Profile.introSource |
| 输入框内容变化 | localStorage | textarea 值 + 当前 AI 意图舱状态 |
| SSE 每条消息接收 | 服务端 DB | 对话消息 + 实时预览内容 |

**草稿恢复流程**：

1. 用户重新访问 `/create/quick` 或 `/create/interview`
2. 前端检查 localStorage 和服务端 draft 状态
3. 如有未完成草稿：
   - 显示提示条：「你有未完成的创建，是否继续？」
   - [从头开始] [继续编辑]
4. 选择继续：恢复输入内容、标签、内容块、对话历史

---

### 3.7 SSE 断线恢复 UX — BP7 修复

> **当前问题**：重试机制存在但恢复后的对话续接体验不清。

**恢复流程**：

```
SSE 连接中断
  │
  ├── 前端自动重试（3次，间隔2s）
  │     └── 显示「正在重新连接...」（琥珀金，AI 进度条样式）
  │
  ├── 重试成功
  │     └── 从最后收到的消息 ID 续接
  │     └── 新消息正常追加，已接收内容不丢失
  │     └── Toast：「连接已恢复」
  │
  └── 3次重试失败
        └── 显示「网络连接中断」提示条
        └── 提供两个操作：
              ├── [重试连接] → 手动触发重连
              └── [保存并稍后继续] → 草稿保存 → 跳转 /profile/[id]
        └── 已接收的内容块和标签全部保留
```

**关键原则**：已接收的任何 AI 输出都不丢失，即使用户选择"稍后继续"。

### 3.8 个人主页状态视觉区分 — BP9 修复

> **当前问题**：仅有空状态定义，缺少 draft/active/dirty 的视觉区分。

| Profile 状态 | 导航栏状态 | 页面顶部提示 |
|-------------|-----------|------------|
| `empty` | 「创建我的主页」CTA | 空状态占位（已定义） |
| `draft` | 「继续创建」(琥珀金描边) | 「你的主页尚未发布」提示条 + 「继续创建」按钮 |
| `active` | 正常品牌名 | 无额外提示 |
| `dirty` | 品牌名 + 琥珀金圆点通知 | 「你有 N 条 AI 更新待确认」提示条 |

**dirty 态提示条样式**：

```css
.profile-dirty-banner {
  display: flex;
  align-items: center;
  gap: 8px;
  padding: 10px 16px;
  background: var(--color-accent-surface);  /* #FDF6EC */
  border: 1px solid var(--color-accent);
  border-radius: var(--radius-md);
  margin-bottom: var(--space-xl);
  font-size: 14px;
  color: var(--color-accent);
```

## 10. 文案规范

| 场景 | 当前文案 | 优化后文案 | 理由 |
|------|---------|-----------|------|
| 创建完成 | 无 | 「你的主页已上线」 | 完成感 + 明确状态 |
| AI 增量待确认 | 无 | 「你有 {N} 条 AI 更新待确认」 | 信息明确 + 行动引导 |
| 新增内容块 | 无 | 「试试用键盘语音输入，口述你想添加的内容」 | 统一语音输入引导 |
| 内容块保存 | 无 | 「已记录，AI 正在补充相关内容」 | 告知后台行为 |
| 草稿恢复 | 无 | 「你有未完成的创建，是否继续？」 | 尊重用户选择 |
| 路径切换 | 无 | 「当前采访进度将被保留为草稿，你可以随时回来继续」 | 减少焦虑 |
| SSE 断线 | 「正在重新连接…」 | 「连接中断，正在重新连接…」 | 更准确的状态描述 |
| 图片生成中 | 无 | 「图片正在生成中，稍后自动更新」 | 设定预期 |
| Profile 尚未发布 | 无 | 「你的主页尚未发布，其他校友暂时看不到」 | 明确后果 |

---

> **核心原则**：每个用户操作都必须有明确的起点、过程和终点。不留悬念，不悬空，不让用户问"然后呢"。
>
> **用户是 Boss**：不仅在 AI 增量内容层面，更在整个产品流程层面 — 用户随时可以保存草稿、切换路径、中断恢复。
>
> **克制而体面**：新增交互不增加页面跳转，全部在原位完成。庆祝状态用文字和微动效，不用 confetti 和弹窗。
